01 / The Discovery

A free iCloud account walked through every door Apple thought it had locked.

Background

In January 2026, while exploring Apple's News Publisher portal at icloud.com/newspublisher, Amro Alomari noticed something strange about how Apple's backend validated who could become a publisher.

The server sent a JSON object — dsInfo — to the browser, containing entitlement flags like isPaidDeveloper and hasICloudQualifyingDevice. The frontend used these flags to decide whether to let the user proceed.

So Amro intercepted the response with Burp Suite, flipped a single boolean — false to true — and pressed continue. What happened next wasn't supposed to be possible.

02 / The Mechanism

Four steps. One boolean.
A complete bypass.

The vulnerability is textbook CWE-602: Client-Side Enforcement of Server-Side Security. Apple's frontend held the trust. Apple's backend asked no questions.

Chapter I

The eligibility check returns trust.

When a user lands on the News Publisher portal, the client requests an eligibility object from the server. This is the moment authorization should happen — and where it quietly doesn't.

GET /setup/ws/1/validate
200 OK 
{
  "dsInfo": {
    "isPaidDeveloper": false,
    "hasICloudQualifyingDevice": true,
    "ContinueOnDeviceEligibleDeviceInfo": { ... }
  }
}
Chapter II

A single byte changes everything.

Using Burp Suite's Match and Replace rule, the response is rewritten before the browser sees it. false becomes true. The frontend now believes — incorrectly — that this is a paid Apple Developer with a qualifying device.

The server is never asked again.

Chapter III

The backend confirms what it never verified.

The bypassed user proceeds through onboarding. A POST to /v1/startup returns a response that should not exist for an unauthorized account:

POST /v1/startup
200 SUCCESS 
{
  "signed": true,                // EULA legally executed
  "migratedToCloudKit": true,    // Database state changed
  "memberships": [{
    "roles": [
      "ADMIN",            // Complete access
      "EDITOR_IN_CHIEF",
      "EDITOR",
      "ISSUE_EDITOR",    // Undocumented role
      "SALES",
      "VIEWER"
    ]
  }],
  "status": { "type": "SUCCESS" }
}
Chapter IV

Apple sends an email to confirm.

Within seconds, an email arrives from newspublisher@apple.com. Apple welcomes the new publisher. The EULA is signed. The account is provisioned in CloudKit. The bypass is now indistinguishable, from Apple's own systems, from a legitimate registration.
